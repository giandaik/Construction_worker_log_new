### 3. Critical Security Flaws Identified (via code review)

Three confirmed bugs — none fixed this session. Tracked in fp issue **CWL-gupyodxk**.

**Finding 1: Unsalted SHA-256 password hashing (HIGH severity)**
- Location: `app/api/login/route.ts:9-11, 29-32`
- Passwords hashed with `crypto.createHash("sha256")` — no salt.
- Consequence: rainbow-tableable; SHA-256 is GPU-fast, making offline brute force trivial on any DB leak.
- Fix: replace with bcrypt (already in `package.json` at line 24 as `bcryptjs@3.0.3` — currently zero imports).

**Finding 2: `bcryptjs` installed but completely unused**
- `bcryptjs@3.0.3` is declared in `package.json` but grep finds zero imports across the codebase.
- This is the correct library to switch to; it just needs to be wired up.

**Finding 3: `POST /api/users` broken — users created without passwords (HIGH severity)**
- `lib/schemas/userSchema.ts:7-11` has no `password` field.
- `lib/repositories/UserRepository.ts:14-21` `User` interface has no `password` field.
- The repository writes directly to the Mongo collection, bypassing the Mongoose `User` model where `password` is `required: true`.
- Result: API-created users have no password stored and cannot log in.

**Secondary issues (not blocking)**:
- `middleware.ts:44-46` returns JSON for page routes on JWT failure instead of redirecting to `/login`.
- `GET /api/users` exposes all user records to any authenticated caller with no role check.
